# Vision — Media OS

## Что такое Media OS

Media OS — личная рабочая система для управления медиа-проектами и текстовой работой. Это не таск-менеджер и не CRM. Это Cockpit: место, где документ, его материалы и шаги работы собраны в одном контексте.

Слоган проекта: **«Человек и ИИ: разжёванное сложное»**.

## Для кого

Media OS создаётся **для одного пользователя** — автора проекта. Это личный инструмент, не командная платформа.

Доступ защищён через Supabase Auth. `user_id` используется только технически для RLS и не является частью пользовательской модели.

## Какие проблемы решает

- **Разрозненность.** Документ, материалы и шаги работы живут в одном месте, а не в заметках, таблицах и таск-трекере одновременно.
- **Старт работы.** Workflow Template предлагает типовой набор действий для типа документа — без обязательной привязки к шаблону после генерации.
- **Фокус внимания.** Today собирает действия из разных документов, которые пользователь сознательно выбрал для текущей работы.
- **Простота.** Минимум сущностей, минимум статусов, минимум визуального шума.

## Чем Media OS не является

- Не проектный менеджер с дедлайнами, Kanban и командой.
- Не система статусов задач («требует решения», «застряло» и т.п.) — эта модель legacy.
- Не AI-редактор и не CMS.
- Не файловое хранилище (Storage не используется; материалы хранят URL/путь как текст).
- Не Notion-клон: Notion — визуальный референс, не целевая функциональность.
